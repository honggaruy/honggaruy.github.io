Enviroment Variables and Shell Variables

  • Variables는 다음과 같은 형식을 갖는다 
    

    1
2
3

    		

    KEY=value
KEY="Some other value"
KEY=value1:value2
    • variables의 이름은 대소문자를 구분한다.
      • 관습적으로, 환경변수는 대문자 이름을 갖는다
    • 다중 값을 할당할때 colon(:)으로 구분한다
    • equal symbol (=) 주변에는 공백을 제거한다
  • Variables 는 두 개의 메인 카테고리로 구분된다
    • Environment variables
      • system-wide하게 적용되는 variable
      • 모든 파생된(spawned) child process와 shell에 상속(inherited)된다
    • Shell variables
      • 현재 shell instance에만 적용된다
      • zsh나 bash 같은 shell은 자신만의 내부 shell variables set을 갖는다
  • Linux에서 환경변수를 다루는 몇가지 commands가 있다
    • env
      • 현재 환경을 바꾸지 않고 custom environment에서 다른 프로그램을 실행할수 있도록 해준다
      • argument 없이 실행하면 현재 환경 변수 목록을 print한다
    • printenv - 모든 지정된 환경변수를 프린트한다
    • set
      • shell 변수를 set하거나 unset한다
      • argument 없이 사용된다면
        • 모든 변수의 목록을 print 한다
        • 모든 변수에는 environment 변수, shell 변수, shell functions까지 포함된다
    • unset - shell 변수와 Environment 변수를 제거한다
    • export - 환경변수를 설정한다

List Environment Variables

  • 환경 변수를 보는데 가장 많이 쓰이는 명령은 printenv 이다
    • 변수이름을 argument로 넘기면 그 변수만 볼수있다
    • argument가 지정되지 않으면 모든 변수가 한 줄에 한 개씩 print된다
    • 사용례 
      

      1
2
3
4
5

      		

      $ printenv HOME
/home/linuxise
$ printenv LANG HOME
en_US
/home/linuxise
  • 가장 흔한 환경 변수들 목록
    • USER - 현재 로그인한 사용자
    • HOME - 현재 사용자의 home directory
    • EDITOR - 기본 편집기. terminal에서 edit을 치면 실행될 편집기이다
    • SHELL - 현재 사용자의 shell 경로. 보통 bash나 zsh
    • LOGNAME - 현재 사용자의 이름
    • PATH - 실행 명령이 탐색될 directory들의 목록
      • 명령을 치면 이 목록에 저장된 순서 명령을 탐색하여 가장먼저 찾은 실행파일을 실행한다
    • LANG - 현재 locales 설정
    • TERM - 현재 terminal emulation
    • MAIL - 현재 사용자의 메일이 저장되는 장소
  • printenv와 env 명령은 환경변수만 print 한다
  • Environment,shell 변수를 포함한 모든 변수와, shell functions 목록까지 보고싶다면
    • set 명령이 있다
    • 이 명령은 매우 많은 량의 목록을 보여주기 때문에 요약판을 보고 싶을 경우 출력을 less 명령으로 pipe한다 
      

      1

      		

      $ set | less
  • echo 명령도 shell 변수값을 print하는데 사용한다 
    

    1
2

    		

    $ echo $BASH_VERSION
5.0.17(1)-release
